## Технології

- NestJS 11
- Node.js 22
- MongoDB + Mongoose
- JWT аутентифікація
- Docker Compose

## Як запустити

```bash
docker compose build
docker compose up
```

API буде доступний на `http://localhost:3000`
Swagger документація на `http://localhost:3000/api/docs`.
Тестовий користувач для логіну: `test@example.com` `+380123456789`

## Ендпоїнти

Всі ендпоїнти вимагають JWT токен окрім `/auth/login:

- `POST /api/v1/auth/login` — отримання JWT токенів
- `POST /api/v1/add-user` — створення нового користувача
- `GET /api/v1/get-users` — список користувачів з пагінацією та фільтрами (name, email, phone)
- `GET /api/v1/get-user/:id` — отримання користувача за ID

## Генерація користувачів

При старті додатка перевіряється кількість користувачів у базі. Якщо база порожня:
1. Створюється один тестовий користувач для аутентифікації
2. Запускається фонова задача, яка генерує 2 мільйони користувачів пакетами по 5000
3. API залишається доступним під час генерації

## Архітектурні рішення

**Пагінація:** cursor-based пагінація через `_id`, щоб уникнути проблем з offset/limit на великих датасетах.

**Індекси:** створено індекси на `email` (unique), `phone` (unique) та `name` для швидкого пошуку.

**Аутентифікація:** JWT токени з refresh механізмом. Логін через email + phone (без пароля).
